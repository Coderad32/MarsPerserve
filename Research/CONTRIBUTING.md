# Contributing to MarsPreserve

Thank you for considering contributing to the MarsPreserve project! Your help is greatly appreciated. To make the process as smooth as possible, please follow these guidelines.

## How to Contribute

### Reporting Issues
If you find a bug or have a feature request, please [open an issue](https://github.com/Coderad32/MarsPerserve/issues) and provide as much detail as possible.

### Submitting Pull Requests
1. [Fork]() the [repository]() and [create]() your [branch]() from [`main`]().
2. [Make]() your [changes]() and [ensure]() that your [code]() follows [the project's coding style]().
3. [Write]() [clear]() and [concise]() commit [messages.]()
4. [Test]() your [changes]() thoroughly.
5. [Submit]() a [pull request]() and describe [the changes you made]().

### Code Style Guidelines

**Shell** Command Line Interpreter
**Python** Functionality Compiler
**Perl** Just A Mess You Clean It Up

Or any other language you may know. show your support and write source code 
for us.

### Branch Naming
Use descriptive branch names that reflect the purpose of the branch, such as:
- `feature/add-new-climate-model`
- `bugfix/fix-temperature-bug`
